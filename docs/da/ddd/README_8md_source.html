5 ## Protocol
6 
7 This section describes the protocol used in the connunication between the server
8 and its clients.
9 
10 
11 ### Common part
12 Each package CAN have the `id` field. This is a random number to identify this
13 package in the session. The other side of the connection can send a packge with
14 the same `id` to indicate, that it's a response to the previous one.
15 
16 
17 ### Client -> Server
18 
19 This section describes packages send by a client.
20 
21 #### JOIN package
22 
23 This package is a join request to the server. A Client can be only in one game
24 at a time. If this packge was sent and the client has already joined to a game,
25 the server will close the connection imediately.
26 
27 ```json
28 {
29  "id": 0,
30  "type": "join",
31  "game": "<the game's name>",
32  "nick": "<player's nick>"
33 }
34 ```
35 
36 ##### Server's Response
37 
38 The `result` field can have to values. `joined` means that, the player has
39 joined to the game. `full` value means that the requested game is full and
40 joining to it is not possible. The `players` field is an array of objects. Each
41 object describes one player. The `rays` field is an array of light rays presnt
42 in the game at the current moment. It can be empty if there are no rays.
43 
44 * The client HAS TO save the values of `player_id` fields, all further updates
45  about players will contain `player_id` field to identify them.
46 * The client HAS TO save the values of `ray_id` fields, all further updates
47  about rays will contain `ray_id` field to identify them.
48 * The `my_id` field's value is the id of this client.
49 
50 **Note**: The `players` and `rays` arrays have meaning only if the joining was
51 successful, otherwise there won't be included and therefore should not be looked
52 for.
53 
54 ```json
55 {
56  "id": 0,
57  "result": "joined|full",
58  "my_id": 1337,
59  "players": [
60  {
61  "player_id": 0,
62  "team_id": 0,
63  "nick": "<player's nick>",
64  "role": "<player's role>",
65  "color": [255, 255, 255],
66  "health": 100.0,
67  "position": [7.6, 87.2],
68  "angle": 67.2 // <0.0; 360.0)
69  },
70  // ...
71  ],
72  "rays": [
73  {
74  "ray_id": 0,
75  "source": [7.6, 87.2],
76  "destination": [17.6, 187.2],
77  "color": [255, 255, 255]
78  },
79  // ...
80  ]
81 }
82 ```
83 
84 #### UPDATE package
85 
86 This package is used to update the position of the player. It should be sent
87 each time when the player either moved, rotated or fired.
88 
89 ```json
90 {
91  "type": "update",
92  "team_id": 0,
93  "position": [7.6, 87.2],
94  "angle": 67.2 // <0.0; 360.0)
95 }
96 ```
97 
98 ##### Server's Response
99 
100 There is no "response" from the server, however the client should not perform
101 any updates on its own sprite, until it receives an UPDATE package from the
102 server, which will be broadcasted to all players.
103 
104 #### LEAVE package
105 
106 This package is used to leave the current game. This request to the server is
107 always successful, therefore the server won't repond to it. The `leave` field
108 can have any value.
109 
110 ```json
111 {
112  "type": "leave"
113 }
114 ```
115 
116 ##### Server's Response
117 
118 **Note**: There is no server's response for this request.

121 ### Server -> Client
122 
123 This section describes the packages send by the server.
124 
125 ### UPDATE package
126 
127 The `players` array contains players objects present in the game, that have
128 updated their attributes. If no player has updated its attributes, the array
129 will be empty, but will not be omitted.
130 If a player either has joined or left, the player object will contain additional
131 fields, described below.
132 The values of `joined` and `left` fields have no meaning and should not be
133 checked.
134 
135 The `rays` array contains rays objects of the game.
136 The values of `new` and `remove` fields have no meaning and should not be
137 checked. If a ray object has neither `new` nor `remove` field it means that, an
138 existing ray has been changed. If the game doesn't have any rays or the existing
139 rays didn't change, the array will be empty, but will not be omitted.
140 
141 ```json
142 {
143  "type": "update",
144  "players": [
145  {
146  "player_id": 1,
147  "position": [0.0, 0.0],
148  "angle": 67.8, // <0.0; 360.0)
149  "health": 100.0,
150  "joined": true, // only when joined
151  "nick": "<new_nick>", // only when joined
152  "role": "<new_role>", // only when joined
153  "color": [255, 255, 255], // // only when joined
154  "left": true // only when left
155  },
156  // ...
157  ],
158  "rays": [
159  {
160  "ray_id": 1,
161  "source": [0.0, 0.0],
162  "destination": [0.0, 0.0],
163  "new": true, // only when a ray is a new one
164  "remove": true // only when a ray should be removed
165  },
166  // ...
167  ]
168 }
169 ```
